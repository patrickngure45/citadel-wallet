## Core Principle

**TST is not an investment. TST is access to capability.**

Users don't acquire TST to hold it. They acquire TST because the system requires it to do something they want.

---

## What TST CAN Do

‚úÖ **Gate Access**
- Creating P2P agreements requires TST lock
- Advanced entity compute requires TST stake
- Priority execution windows require TST hold

‚úÖ **Enable Behavior**
- Higher capital limits at higher TST stakes
- Faster execution windows (reduced cooldowns)
- Advanced strategy parameters
- Override restrictions in edge cases

‚úÖ **Secure Commitment**
- P2P agreement collateral (locked for deal duration)
- Entity compute reservation (locked during simulation)
- Risk override guarantee (locked until action completes)

‚úÖ **Access Tier System**
- Tier 0: No TST required (basic deposits, simple strategies)
- Tier 1: 10 TST locked (P2P creation, priority windows)
- Tier 2: 50 TST locked (advanced entities, higher limits)
- Tier 3: 250+ TST locked (institution-grade features, custom parameters)

‚ùå **Promise Yield**
- No "8% APY"

‚ùå **Be Marketed as an Investment**
- Never position as "early opportunity"

- Never use language like "initial token sale"
- Never compare to other token projects

‚ùå **Inflate Passively**

‚ùå **Be Easily Tradeable**
- No listing on retail DEXs during Phase 1-2

‚ùå **Fund Development**
- No token sale to bootstrap company
- No "treasury" that dilutes holders

## Decision Record

**Why this narrative?**

1. **Survivability** - Tokens with clear mechanics survive. Tokens with yield promises die.
2. **Network Quality** - Friction attracts serious users. Hype attracts speculators.
3. **De-risking** - If TST fails, it's because the product failed, not the tokenomics.
4. **Defensibility** - You can't be criticized for "over-promising" if you promised nothing.

**Why lock it now?**

Because the pressure to deviate will be intense once money is involved. Freezing it now (before code ships) means you're not fighting yourself later.
